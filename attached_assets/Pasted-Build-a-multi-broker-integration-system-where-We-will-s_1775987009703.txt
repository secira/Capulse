________________________________________AUTHENTICATION MODEL (CRITICAL)
✅ Use: OAuth / Token-Based Login
❌ DO NOT:
•	Store user passwords
•	Scrape broker UI
________________________________________
🔄 Standard Flow
Step 1: User clicks “Login using  Broker”. See the Sample screen attached from Sensibull. 
Frontend → Backend → Broker Auth URL
________________________________________
Step 2: Redirect to Broker Login
Example (Zerodha):
https://kite.zerodha.com/connect/login?v=3&api_key=YOUR_API_KEY
________________________________________
Step 3: Broker redirects back with request_token
https://yourapp.com/callback?request_token=XYZ
________________________________________
Step 4: Exchange token
Backend:
access_token = exchange_request_token(request_token)
________________________________________
Step 5: Store securely
{
  "broker": "zerodha",
  "access_token": "...",
  "expires_at": "...",
  "user_id": "..."
}
